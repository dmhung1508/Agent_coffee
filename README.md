# Coffee Multi-Agent (8AM Coffee)

Production-grade Vietnamese coffee shopping agent. LangGraph multi-agent assistant cho cửa hàng cà phê 8AM, wrap public read-only menu API tại `https://api-coffee.8am.vn`. Hỗ trợ tư vấn món, quản lý giỏ hàng, sinh QR thanh toán VietQR, expose qua CLI streaming và FastAPI HTTP/SSE.

Inspired by — but not copying — NVIDIA's [Retail Shopping Assistant](https://github.com/NVIDIA-AI-Blueprints/retail-shopping-assistant).

## Architecture

```
START
  │
  ▼
fast_path_node ──(greeting/thanks/goodbye)──► summary_node ──► END
  │
  ▼
memory_node
  │
  ▼
planner_node
  │
  ├──► retriever_node ──► chatter_node ──┐
  ├──► cart_node ─────────────────────────┤
  ├──► checkout_node ─────────────────────┤
  ├──► chatter_node ──────────────────────┤
  ├──► unsupported_node ──────────────────┤
  └──► error_node ────────────────────────┤
                                          ▼
                                    summary_node ──► END
```

Key features (from the productization spec in `.kiro/specs/coffee-agent-quality-fix/`):

- Fast-path bypass for greeting / thanks / goodbye — zero LLM cost
- Token-level streaming via `graph.astream_events` (CLI + SSE)
- TTL+LRU menu cache (`cachetools`) with per-key invalidation
- Lazy enrichment for `browse_menu` (top-N, default 3)
- `last_catalog` invalidation on topic shift (Jaccard overlap < 0.3)
- UUID4 `order_id` per checkout with append-only JSONL audit log
- Structured JSON logging (`structlog`) + optional LangSmith tracing
- Cart deduplication by id or `(fold_text(name), type)` keyed
- Ambiguous-remove confirmation (≥2 substring matches)
- Grounded-only chatter prompt — no hallucinated menus or prices

## Setup

```powershell
cd F:\agent_new
python -m pip install -r requirements.txt
Copy-Item .env.example .env
# Edit .env to set OPENAI_API_KEY (the only required var)
```

`PYTHONIOENCODING=utf-8` is set automatically inside the CLI. The full env var matrix is in [.env.example](./.env.example) and the table below.

## Run

### Interactive CLI (token streaming, default)

```powershell
python coffee_multi_agent.py
# or explicitly
python coffee_multi_agent.py cli
# debug mode (node lifecycle + final timing summary)
python coffee_multi_agent.py cli --debug
```

Backward-compat: legacy `python coffee_multi_agent.py --debug` (no subcommand) still works — argparse injects `cli` when no known subcommand is present.

Sample prompts:

```text
Gợi ý cà phê dễ uống
Tìm cà phê muối
Cho tôi xem chi tiết cà phê muối
Thêm món đầu tiên vào giỏ
Xem giỏ hàng
Tổng giỏ hàng bao nhiêu
Chốt đơn
```

### HTTP server

```powershell
python coffee_multi_agent.py serve --host 0.0.0.0 --port 8000
# or via uvicorn directly
uvicorn coffee_agent.server:app --host 0.0.0.0 --port 8000
```

`--host` and `--port` fall back to `HTTP_HOST` / `HTTP_PORT` from settings when omitted. Add `--reload` for dev autoreload.

## API endpoints

### `POST /chat`

Single-turn request/response. Carry the `session_id` returned in the first response to keep cart state across turns.

```bash
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"query":"menu có gì"}'
```

Response (truncated):

```json
{
  "session_id": "abc123...",
  "turn_id": "...",
  "final_answer": "Quán hôm nay có cà phê muối, bạc xỉu, ...",
  "cart": [],
  "cart_total": null,
  "order_stage": "browsing",
  "order_id": null,
  "route": "retriever",
  "timings_ms": {"memory": 0, "planner": 320, "retriever": 180, "chatter": 410, "summary": 5}
}
```

### `GET /chat/stream` (SSE)

Server-Sent Events stream — emit chatter tokens as they're generated by the LLM.

```bash
curl -N "http://localhost:8000/chat/stream?query=menu+co+gi&session_id=abc123"
```

Event sequence:

```
event: node_start    data: {"node":"retriever_node"}
event: node_end      data: {"node":"retriever_node"}
event: node_start    data: {"node":"chatter_node"}
event: token         data: {"text":"Quán "}
event: token         data: {"text":"hôm nay "}
...
event: node_end      data: {"node":"chatter_node"}
event: final         data: {<full ChatResponse JSON>}
```

### `GET /healthz`

```bash
curl http://localhost:8000/healthz
```

```json
{"status":"ok","uptime_s":1234,"version":"0.2.0","menu_api":"reachable","cache_size":18,"sessions":4}
```

Returns `503` with `status: "degraded"` after 3 consecutive menu-API failures.

Full list (defaults shown). See [.env.example](./.env.example) for the file template.

|-----|---------|---------|

| `COFFEE_API_BASE_URL` | `https://api-coffee.8am.vn` | Public menu API base |

| `MENU_CACHE_TTL_SECONDS` | `600` | Menu cache freshness |

| `BROWSE_ENRICH_TOP_N` | `3` | Lazy detail enrichment count |
| `LAST_CATALOG_OVERLAP_THRESHOLD` | `0.3` | Topic-shift sensitivity |
| `LANGSMITH_API_KEY` | (optional) | LangSmith tracing |
| `LANGCHAIN_TRACING_V2` | `false` | Enable LangSmith |
| `LOG_LEVEL` | `INFO` | structlog level |
| `LOG_JSON` | `true` | JSON vs console renderer |
| `FAST_PATH_ENABLED` | `true` | Skip LLM for greetings |
| `SESSION_TTL_SECONDS` | `3600` | Session eviction window |

| `HTTP_PORT` | `8000` | Server bind port |
| `CORS_ALLOWED_ORIGINS` | `*` | Comma-separated origin list |

## Logs

Structured JSON to stdout (one record per log call):

```json
{"timestamp":"2026-04-29T10:00:01.234Z","level":"info","event":"node_end",
 "node":"retriever_node","turn_id":"e3b0c442","session_id":"ab12...",
 "next_agent":"retriever","latency_ms":312,"api_endpoint":"GET /public/v1/menu",
 "api_item_count":5}
```

Order audit log (`logs/orders.jsonl`, append-only, OS-level locked):

```json
{"order_id":"abc123...","session_id":"...","items":[{"id":"cpm","name":"Cà phê muối","type":"dish","price":29000,"quantity":2}],"total":58000,"qr_url":"https://img.vietqr.io/image/MB-669699669-compact.png?amount=58000","created_at":"2026-04-29T10:00:01.234Z"}
```

Set `LOG_JSON=false` to switch to a console-friendly renderer for local development.

## Tests

```powershell
python -m pytest tests/ -v
```

The suite contains 130+ tests across:

- `tests/test_bug_conditions.py` — 17 PBT exploration cases mapping 1-1 to defects E1–E17 (design 3.2). MUST FAIL on the unfixed `main` branch — passing here means the bug is gone.
- `tests/test_smoke.py` — 7 end-to-end integration scenarios (design 8.17): greeting fast-path, browse menu, search, ordinal add, single-match remove, ambiguous remove, checkout with `order_id`.
- `tests/test_text.py`, `test_cache.py`, `test_fast_path.py`, `test_menu_client.py`, `test_state.py`, `test_session_store.py`, `test_order_log.py` — per-component unit tests (design 12.5).
- `tests/test_properties.py` — Hypothesis-driven property tests for ambiguous-remove, dedup, invalidation, fast-path safety (design 12.6).
- `tests/test_server.py` — FastAPI test-client coverage of `/chat`, `/chat/stream`, `/healthz`, session persistence (design 12.7).

## Project structure

```
coffee_agent/
  __init__.py
  agents/        # MemoryNode, PlannerAgent, RetrieverAgent, CartAgent,
                 # ChatterAgent, CheckoutAgent, SummaryAgent, UnsupportedAgent
  cache.py       # MenuCache (TTL+LRU)
  errors.py      # CoffeeAgentError hierarchy
  fast_path.py   # Regex-based social message detector
  formatting.py  # Cart/catalog rendering
  graph.py       # LangGraph factory
  logging_config.py  # structlog setup + logged_node decorator
  menu_client.py # PublicMenuClient with retry/backoff
  order_log.py   # JSONL order persistence
  prompts.py     # PLANNER_SYSTEM, CHATTER_SYSTEM, few-shots, context builders
  runtime.py     # stream_turn / run_turn helpers
  server.py      # FastAPI app
  session_store.py  # TTL session store
  settings.py    # Pydantic Settings
  state.py       # CoffeeState, Cart, CartItem, TurnRecord, OrderRecord
  text.py        # fold_text, keyword_overlap

coffee_multi_agent.py  # CLI entry (cli/serve subcommands)
tests/                 # pytest suite
```

## Architecture credits

The pipeline shape (memory → planner → specialists → chatter → summary) is inspired by NVIDIA's [Retail Shopping Assistant blueprint](https://github.com/NVIDIA-AI-Blueprints/retail-shopping-assistant), adapted to a single-process LangGraph deployment instead of the original microservice stack.
